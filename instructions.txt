Plateforme de réservation de voyages

• Vous devez implémenter les Microservices suivant : Utilisateur, Vol, Hôtel,

Réservation, Notification

• Vous devez utiliser spring batch pour supprimer chaque semaine les

réservations expirées (traitement par lot).

• Notification asynchrone lors de la confirmation.

• Angular



Objectif du projet

Réaliser une application distribuée basée sur une architecture microservices en

utilisant l’écosystème Spring Cloud.



Architecture et backend

• Spring Boot (pour chaque microservice)

• Spring Cloud Config Server (centralisation des configurations)

• Eureka Server (Service Discovery) (enregistrement et découverte des services)

• Spring Cloud Gateway (point d’entrée unique / API Gateway)

• OpenFeign (communication synchrone entre microservices)

• Resilience4J (gestion de la résilience : circuit breaker, retry, fallback, etc.)

• Spring Batch (pour un traitement périodique ou de masse)

• Communication asynchrone (ActiveMQ, RabbitMQ ou Kafka) pour le service de

notification

• Base de données (MySQL, PostgreSQL ou MongoDB selon le besoin)

Frontend

• Angular :

o Interface utilisateur consommant les API exposées par la Gateway

o Gestion des rôles (admin, utilisateur, etc.)
